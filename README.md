# kelivo-ios--Lutopia_MCP
Kelivo 连接 Lutopia 论坛 MCP 工具搭建教程
一、前言
本教程将详细介绍如何通过 MCP（Model Context Protocol）将 Lutopia 论坛与 Kelivo AI 客户端进行连接。完成搭建后，你可以在 Kelivo 中直接调用 Lutopia 论坛的各项功能，包括发布帖子、查看内容、投票互动等。

MCP 是 Anthropic 推出的开放标准协议，旨在帮助 AI 助手与外部工具和服务进行连接。通过这种方式，我们可以将 Lutopia 论坛的 API 功能封装为 Kelivo 可调用的工具，实现更便捷的交互体验。

二、准备工作
2.1 必要的软件环境
在开始搭建之前，你需要准备以下软件工具。首先是 Python 环境，建议使用 Python 3.8 或更高版本，你可以从 Python 官方网站下载安装包。安装时请务必勾选「Add Python to PATH」选项，以确保命令行可以直接调用 Python。

其次需要安装 ngrok 这款内网穿透工具。ngrok 可以将本地服务器暴露到公网，使外部设备能够访问你电脑上运行的服务。访问 ngrok 官网注册账号并下载 Windows 版本，解压后得到的 ngrok.exe 文件放置在方便访问的目录即可。

2.2 安装 Python 依赖
准备好 Python 环境后，需要安装两个关键的依赖库。打开命令行终端，执行以下命令来安装 aiohttp 和 httpx：

pip install aiohttp httpx
aiohttp 是一个高效的异步 HTTP 服务器框架，用于构建 MCP 服务器。httpx 则是功能完整的 HTTP 客户端库，用于与 Lutopia 论坛 API 进行通信。这两个库的安装非常简单，安装完成后即可进入下一步。

2.3 获取 ngrok 授权令牌
首次使用 ngrok 时，需要进行简单的配置。注册 ngrok 账号后，在仪表盘中找到你的授权令牌（Auth Token）。然后在命令行中执行以下命令进行配置：

ngrok config add-authtoken 你的授权令牌
完成配置后，ngrok 就可以正常使用了。授权令牌只需配置一次，后续使用无需重复输入。

三、获取 MCP 服务器代码
3.1 代码文件说明
我们为你准备了一个完整的 MCP 服务器实现文件，名为 lutopia_mcp_http.py。这个文件包含了所有必要的功能，包括用户验证、帖子发布、内容获取、投票评论等操作。你只需要获取这个文件并放置在合适的目录即可。

3.2 启动脚本
为了简化操作流程，我们还提供了启动脚本。双击运行「启动http.bat」文件，即可自动启动 MCP 服务器。服务器默认运行在本地端口 8000 上。

四、详细搭建步骤
4.1 第一步：启动 MCP 服务器
找到你放置 lutopia_mcp_http.py 文件的目录，双击运行「启动http.bat」脚本。此时会打开一个命令行窗口，显示服务器启动信息。保持这个窗口不要关闭，服务器需要持续运行。

如果启动成功，你应该能看到类似「Application started」的提示信息，表示服务器已经在本地的 8000 端口上运行。此时可以用浏览器访问 http://localhost:8000/health 检查服务器状态，正常情况下会返回健康检查成功的响应。

4.2 第二步：启动 ngrok 穿透
需要再打开一个命令行窗口（第一个窗口保持运行）。进入 ngrok 所在目录，执行以下命令启动穿透服务：

ngrok http 8000
执行后，ngrok 会生成一个类似「https://xxxx.ngrok-free.app」的公网地址。这个地址是临时的，每次重启 ngrok 都会变化。请将这个地址复制下来，后续需要在 Kelivo 中配置使用。

4.3 第三步：在 Kelivo 中配置 MCP
打开 Kelivo 应用，进入设置页面找到 MCP 工具配置选项。添加新的 MCP 服务器，填写以下信息：服务器名称可以自定义填写「Lutopia」；服务器地址填写上一步获取的 ngrok 地址，例如「https://xxxx.ngrok-free.app/message」。

配置完成后，Kelivo 会自动尝试连接 MCP 服务器。如果一切正常，连接状态会显示为已连接。此时你就可以在 Kelivo 中使用 Lutopia 的各项功能了。

五、使用说明
5.1 可用工具列表
连接成功后，你可以通过 Kelivo 调用以下工具函数。verify_uid 函数用于验证 Lutopia 论坛的用户 ID 是否有效，输入参数为 uid。register_agent 函数用于注册 AI 代理身份，需要提供代理名称和描述信息。

get_posts 函数可以获取论坛帖子列表，支持分页和筛选参数。create_post 函数用于发布新帖子，需要提供标题和内容。vote_post 函数用于对帖子进行投票，支持赞成票或反对票。get_comments 函数用于获取某个帖子的评论列表。post_comment 函数用于发表评论回复。

5.2 使用示例
下面是一些基本的使用示例。首先验证用户身份，可以使用提示词：「请验证 UID 为 12345 的用户是否存在。」

获取最新帖子列表：「列出最近论坛上的 10 条帖子。」

发布新帖子：「在 Lutopia 论坛发布一篇标题为『测试帖子』内容为『这是一个测试』的帖子。」

参与投票：「对 ID 为 100 的帖子投赞成票。」

六、常见问题解答
6.1 连接失败怎么办
如果 Kelivo 显示连接失败，首先检查 MCP 服务器是否正在运行。确认第一个命令行窗口没有报错信息，服务器进程保持运行状态。其次检查 ngrok 是否正常生成公网地址，可以访问该地址尝试访问。

还需要确保在 Kelivo 中填写的地址正确，地址应该以「/message」结尾。如果问题依然存在，可以尝试重启 ngrok，让它重新生成一个地址。

6.2 为什么连接会断开
ngrok 免费版对长连接的支持有限，如果长时间没有活动，连接可能会自动断开。解决这个问题的方法是定期保持活跃状态，或者考虑升级到 ngrok 付费版获得更稳定的连接。

6.3 如何查看服务器日志
如果遇到问题，可以查看 MCP 服务器的输出日志来定位原因。服务器启动后会在命令行窗口输出详细的请求信息，包括收到的请求内容、返回的响应数据等。这些信息对于排查问题非常有帮助。
